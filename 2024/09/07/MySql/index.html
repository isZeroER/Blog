1. 库操作

1.1.登录和退出数据库

user -uroot -p+password;
exit;

1.2.显示所有数据库

show databases;

1.3.选中一个数据库使用

use database;

1.4.创建数据库/删除数据库

create database [name];
drop database [name];

2. 表操作

2.1.显示数据库中的所有表

show tables;

2.2.显示表中所有数据

select * from [tableName];

2.3.创建数据表

create table [tableName](
[data1] [type],
[data2] [type]);

2.4.查看数据表结构详情

describe [tableName];

2.5.向表中添加数据记录

insert into [name]
values([value1], [value2]…);

  • mysql常用数据类型
  1. 数值
  2. 日期
  3. 字符串

2.6.删除特定一条数据

delete from [tableName] where [data]=[value];

2.7.修改特定数据

update [tableName] set [data]=[value] where [data]=[value];

2.8.特定查询

select * from [tableName] where [data] = [value];

3.约束

3.1.主键约束

create table [tableName] (
[data1] [type1] primary key,
[data2] [type2],
);
主键约束只能有一个，用于唯一标识每一不同的数据

3.2.联合主键

create table [tableName](
[data1] [type],
[data2] [type],
primary key(data1, data2)
);
这样data1和data2都是联合主键，两个与所有的其他数据不完全相同即可

3.3.自增约束

create table [tableName](
[data1] [type] primary key auto_increment,
[data2] [type],
);

3.4.唯一约束，不能重复

create table [tableName] (
[data1] [type1] unique,
[data2] [type2]
unique(data2);
);
联合唯一约束
create table [tableName] (
[data1] [type1]
[data2] [type2]
unique(data1, data2);
);

3.5.非空约束

create table [tableName](
[data1] [type1] not null,
[data2] [type2]
);

3.6.默认约束：如果插入数据的时候没有输入数据，就使用默认值

create table [tableName](
[data] [type] default [value]
);

3.7.外键约束：涉及两个表，父表子表

create table class(
id int primary key,
name varchar(20)
);
create table student(
id int primary key,
name varchar(20),
class_id int,
foreign key(class_id) references class(id)
);
1）student.class_id只能约束为class.id里面出现过的数据
2）主表里面的数据如果已经被子表引用，那么主表里面的数据不能删除

4.表结构操作DDL：

4.1.补充约束

alter table [tableName] add primary key(dataName);
alter table [tableName] add unique(dataName);
如果要添加的dataName有的数据里面已经有了重复，就不能补充成功

4.2.删除主键约束

alter table [tableName] drop primary key;

4.3.修改列数据结构

alter table [tableName] modify [dataName] int primary key;
alter table [tableName] change [dataOldName] [dataNewName] [type] [restraint];

4.4.删除唯一约束

alter table [tableName] drop index [dataName];

4.5.删除整个表数据

delete from [tableName];
truncate table [tableName];

4.6.查看建表语句和字符集

show create table [tableName]

4.7.添加列

alter table [tableName] add [dataName] [type] [restraint];

4.8.修改表名

rename table [tableOldName] to [tableNewName];

4.9.修改表字符集

alter table [tableName] character set [encodeName];

5.数据库三大范式

范式越详尽，对于实际操作可能更好，但不绝对

5.1.第一范式1NF

字段不能继续拆分就ok

5.2.第二范式2NF

在第一范式的前提下，除了主键（或者联合主键）之外的字段都必须完全依赖于主键
这种情况下，就可以将该表拆分为多个互不相干的表

5.3.第三范式3NF

在第二范式的前提下，除了主键外的其他列之间不能有传递依赖关系

6.表查询

6.1.全表查询

select * from [tableName];

6.2.特定列查询

select data1, data2… from [tableName];

6.3.列排重查询

select distinct [dataName] from [tableName];

6.4.查询区间

select * from [tableName] where [data] between min and max;
select * from score where score > 80 and score < 90;
select * from score where score in (85, 86, 88);
可以直接使用运算符比较

6.5.or, and

select * from student where class=’4’ or ssex=’girl’;
select * from score where score > 80 and score < 90;

6.6.升序降序查询记录(默认是升序）

降序
select * from score order by score desc；
升序
select * from score order by score asc;
联合排序，先排前面的条件
select * from score order by cid asc, score desc;

6.7.数量查询

select count(*) from student where class=’3’;

6.8.最高查询

select sid, cid from score where score=(select max(score) from score);
实际上是嵌套查询了

  • 排序搜查：
    select sid, cid, score from score order by score desc limit 0,1;
    0代表起始位置下标，1表示选取个数

6.9.平均查询

select avg([dataName]) from [tableName];
可以添加where查询
还可以分组查询
select avg([dataName]) from [tableName] group by [dataName];

6.10.分组条件和模糊查询

select cid, avg(degree), count(*) from score group by cid
having count(cid)>=2
and cid like’3%’;

分组条件使用group by, having，然后使用like进行模糊查询（正则表达式）

6.11.范围查询两种方式

select * from score where degree > 60 and degree < 80;
select * from score where degree between 69 and 80;

6.12.多表查询

  • 主键的重要性：用以在多个表中进行匹配相等
  • 双表查询
    select cid, sname, degree from student, score where student.sid=score.sid;
    添加一个where查询
    多表查询步骤：先分别查表，然后综合查表
  • 三表查询
    select sname, cname, degree from student, course, score
    where student.sid=score.sid
    and course.cid=score.cid;
    如果要将不同表中的相同数据展示，需要使用as取别名
    select sname, cname, degree, student.sid as stu_sid, score.sid from student, course, score
    where student.sid=score.sid
    and course.cid=score.cid;

6.13.子查询加分组查询平均分

例子1：查询2班学生每门课程的平均分

  • 先把2班学生的sid都查询
    select sid from student where class=2;
  • 再利用in查找score里面的分数
    select * from score where sid in (select sid from student where class=2);
  • 再去求平均分
    select cid, avg(degree)
    from score
    where sid in(select sid from student where class=2)
    group by cid;

例子2：查询选修5472课程的成绩高于103号同学在5472课程上成绩的同学
select * from score
where cid=5472
and degree > (select degree from score where sid=103 and cid=5472);

6.14.year函数与带in关键字的子查询

例子：查询与学号为101和102的同学同年出生的所有学生的sid、sname、birthday
select * from student where year(sbirth) in (select year(sbirth) from student where sid in (101,103));
要使某个值等于多个里面的一个，使用in

6.15.多层嵌套子查询

select sid, degree from score where cid=(select cid from course where teachernum=(select tid from teacher where tname=’zx’));

6.16.多表查询

例子：查询选修某课程的同学人数多于5人的教师姓名
select tname from teacher
where tid in
(select tid from course where cid in
(select cid from score group by cid
having count(cid)>2));

6.19. union的使用

  • 使用Union可以将两个表拼凑在一起，求并集。默认情况下会去重，如果使用union all就不会去重，因此效率更高
    select * from teacher where tid=801
    union
    select * from teacher where tid=802;

6.20.any 和 all的使用

  • any表示至少，比如：
    select degree from score where degree > any(select degree from score where cid=5472);
    这里any表示大于括号中任意一项即可
  • all表示所有，比如：
    select degree from score where degree > all(select degree from score where cid=5472);
    这里all表示大于所有括号中的值才可以

6.21.as的使用

  • 一般跟随上面部分
    mysql> select tname as name, tsex as sex, birth from teacher
    -> union
    -> select sname, ssex, birth from student;

6.22.复制表用于条件查询

select * from score a where degree<(select avg(degree) from score b where a.cid=b.cid);
a表和b表进行cid匹配，b表负责将平均分计算出来，a表用平均分进行比对

6.23.分组条件查询

  • 需要先把全局条件放在前面，然后再分组，分组后可以进行组条件
    select class_id from student where ssex=’boy’ group by class_id having count(*) >1;

6.24.year函数和now函数

  • year(now())是获取当前的年号，year(birth)获取一个date类型的年份
    select tname, year(now())-year(birth) as old from teacher;

6.25.等级查询

mysql> create table grade(
-> low int(3),
-> upp int(3),
-> grade char(1));
利用between and进行等级划分
select grade from score,grade where degree between low and upp;
这里的low和upp就是直接在每一条里面进行匹配

7.MySQL的四种连接查询

  1. （inner） join（内联查询、内连接)：其实就是将两张表的数据，通过某个字段相对，查询出相关记录数据
    select * from person inner join card on person.cardId=card.id;
    使用on作为条件
  2. left（outer）join（左外连接）:会把左边表里的数据全部显示出来，如果右边表中数据有符合条件就显示出来，如果没有就会补上null
    select * from person left outer join card on person.cardid=card.id;
  3. right （outer）join（右外连接）：会把右边表里的数据全部显示出来，如果左边表中数据有符合条件就显示出来，如果没有就会补上null
    select * from person right outer join card on person.cardid=card.id;
  4. full（outer）join（全外连接）：
    select * from person full outer join card on person.cardid=card.id;
  • 实际上就是左外连接和右外连接的并集
    select * from person left outer join card on person.cardid=card.id;
    union
    select * from person right outer join card on person.cardid=card.id;

8.事务

  • 定义：事务就是一个最小的不可分割的工作单元。事务能够保证一个业务的完整性。例如A用银行卡转账给B，不能只发不收，也不能只收不发
  • 所以在MySQL中，多条语句可能会有同时成功的要求，或者同时失败的要求。
  • MySQL默认情况下是开启事务的,查看事务开启
    select @@autocommit;
    如果开启了事务，那么就不能使用回滚：rollback；撤销sql语句执行效果
  • 关闭事务：set autocommit=0; 就可以使用rollback；进行操作回滚
  • 没有自动commit，那么展示出来的就是一个copy过的虚拟表。如果手动commit；那么就进行提交，rollback；也不能回滚了

8.1.事务的实际作用

  • 如果有多条语句要求是同时作用，其中有一条失败了，我们就可以rollback，成功了就手动commit

8.2.使用事务的方法

  1. 修改默认的autocommit=0.
  2. begin;
  3. start transaction;
  • 2、3都是开启一个事务，在这个事务里面可以进行一些操作，在commit之前都可以一次性rollback。但是一旦commit或者已经rollback之后，就不能再回滚

8.4.事务的四大特性ACID

  • A 原子性：事务是最小的单位，不可以再分割
  • C 一致性：事务要求，同一事务中的sql语句，必须保证同时成功或者同时失败
  • I 隔离性：事务1和事务2之间是具有隔离性的
  • D 持久性：事务一旦结束（commit或者rollback），就不可返回

8.5.事务的隔离性

  • 查看数据库的隔离级别：
    MySQL 8.0：select @@global.transaction_isolation;
    MySQL 5.x：select @@global.tx_isolation;
    会话级别就不加上global
  • 修改隔离级别
    set global transaction isolation level [DLevel];
  1. read uncommitted; 脏读
    如果有事务A对数据进行操作，操作的过程中还没有提交事务，而事务B可以看见事务A操作的数据，也就是脏读。举例：小明骗淘宝店
    实际开发中，不允许脏读出现
  2. read committed; 不可重复读
    如果开启一个事务A去读取数据，然后事务B进行了一些数据的更改，事务A继续去读取就会有不同，也就是不可重复读
  3. repeatable read； 幻读
    事务A和事务B都开启了，事务A和B同时操作一张表，然后A进行数据更改，B不能看到数据的更改。也就是幻读
  4. serializable; 串行化
    事务A和B同时操作一个表的时候，如果A在操作，B就不能操作，进入等待，直到A commit。也就是排队状态（串行化）。问题：性能特别差
  • 性能排比：uncommitted>committed>repeatable read>serializable。默认级别是repeatable read
