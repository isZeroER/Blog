1.基础篇

1.1.渲染流水线

  1. 应用阶段（CPU执行）
  • 准备场景数据：摄像机位置、视锥体、场景中模型、使用的光源；
    粗粒度剔除：把不可见物剔除；
    设置渲染状态：模型所要使用到的纹理、材质、shader
  • 应用阶段将渲染图元传递给几何阶段
  1. 几何阶段（GPU执行）
  • 对渲染图元进行处理，输出屏幕空间的二维顶点坐标、每个顶点对应的深度值、着色等相关信息，传递给光栅化阶段
  1. 光栅化阶段（GPU执行）
  • 根据几何阶段传递的数据进行屏幕像素绘制，渲染出最终图像

1.2.GPU与CPU通信

1.2.1.CPU（应用阶段）

  1. 把数据加载到显存中（HDD-RAM-VRAM）
  2. 设置渲染状态
  3. 调用DrawCall，也就是渲染命令（一般在几十到一百）

1.2.2. GPU流水线（几何阶段和光栅化阶段）

  • 几何阶段：
  1. 顶点着色器（Vetex Shader）：负责进行坐标变换和逐顶点光照
  2. 剪裁（Clipping）：将一些摄像机视野外的场景物体处理掉
  3. 屏幕映射（Screen Mapping）：将每个图元的xy坐标映射到屏幕坐标系，z坐标系不变。
  • 光栅化阶段：计算每个图元覆盖了哪些像素，以及为这些像素计算他们的颜色
  1. 三角形设置（Triangle Setup）：计算光栅化一个三角网格所需信息
  2. 三角形遍历（Triangle Traversal），又称扫描变换（Scan Conversion）：检查每个像素是否被一个三角网格覆盖，是则产生片元（fragment，包含很多状态信息的集合）。还会根据三角形设置输出的三个顶点信息对整个覆盖区域的像素进行插值
  3. 片元着色器（Fragment Shader）：输出一个或者多个颜色值，其中包含纹理采样等渲染技术
  4. 逐片元操作（Per-Fragment Shader）：通过多个测试来决定每个片元的可见性，并且把这个片元的颜色值与已经存储在颜色缓冲区中的颜色进行合并
    a. 模板测试：
    通过模板比较函数，判断模板缓冲区中该片元的模板值与参考值的大小关系来决定是否舍弃
    b. 深度测试：
    通过深度比较函数，判断该片元的深度值与已经存在于深度缓冲区的深度值的大小关系来决定是否舍弃
    c.合并：
    通过a、b测试后，看是否开启混合（Blender）模式，以判断是否混合还是直接覆盖。
  • 一些问题：
  1. 按正常的渲染管线，那么深度测试在管线的末端，如此前期可能有一些片元经过了一些列操作之后，在深度测试没有通过，那么浪费了很多性能
  2. 有一种Early-Z的技术，将深度测试提前到片元着色器之前可以带来很多性能提升，但是这样可能会与片元着色器的一些操作冲突。

1.2.3.CPU与GPU工作

  • CPU将Draw Call以及改变渲染状态命令发送到命令缓冲区，GPU从中读取命令。
  • 实际上GPU的执行速度很快，但是CPU的速度会比较慢，因为多Draw Call涉及到准备数据、状态和命令，还要检查当前渲染状态
  • 减少Draw Call的办法之一：批处理（Batching），可以对静态物体进行一次合并，避免用大量很小的网格，也避免使用过多材质

1.3.Unity Shader

1.3.1.Unity中的Shader

  1. Standard Surface Shader：包含了一个标准光照模型的表面着色器模板
  2. Unlit Shader：不包含光照（但包含雾效）的基本顶点/片元着色器
  3. Image Effect Shader：提供实现各种屏幕后处理效果的基本模板
  4. Compute Shader：利用GPU并行性来进行一些与常规渲染流水线无关的计算

1.3.2.Unity Shader的ShaderLab

  • 基础结构：
  • 表面着色器：与光源有很多交道
  • 顶点/片元着色器：更加灵活，且适合光照比较少的情况

1.4.MVP变换

  • 顶点着色器：负责进行坐标变换，通过矩阵的方式进行变换
  • 变换过程：
  1. 模型空间：也就是局部空间/对象空间
  2. 世界空间：通过模型变换，从模型空间到局部空间
  3. 观察空间：通过观察变换，从世界空间投射到摄像机的观察空间
  4. 裁剪空间：通过投影变换，将观察空间的顶点剪裁到视锥体当中
  5. 屏幕空间：通过屏幕映射将视锥体中点投射到屏幕空间当中
  • 其中只有观察空间，也就是相机空间是右手系，其他都是左手系

2.初级篇

2.1.简单Shader学习

2.1.1.Shader编写

  • 简单实现
  • 定义结构体
    Pass
    {
    CGPROGRAM
    #pragma vertex vert
    #pragma fragment frag

    #include “UnityCG.cginc”

    struct v2f
    {
    float4 pos:POSITION;
    fixed4 color:COLOR0;
    };

    v2f vert(appdata_full v)
    {
    v2f o;
    o.pos = UnityObjectToClipPos(v.vertex);

    //可视化法线
    o.color = fixed4(v.normal*.5 + fixed3(.5, .5, .5) , 1);
    //可视化切线
    o.color = fixed4(v.tangent.xyz * .5 + fixed3(.5, .5, .5), 1);
    //可视化副切线
    fixed3 binormal = cross(v.normal, v.tangent.xyz) * v.tangent.w;
    o.color = fixed4(binormal * .5 + fixed3(.5, .5, .5), 1);
    //可视化第一组纹理坐标
    o.color = fixed4(v.texcoord.xy, 0, 1);
    //可视化第二组纹理坐标
    o.color = fixed4(v.texcoord1.xy, 0, 1);
    //可视化第一组纹理坐标的小数部分
    o.color = frac(v.texcoord);
    if(any(saturate(v.texcoord) - v.texcoord))
    {
    o.color.b = .5;
    }
    o.color.a = 1;
    //可视化第二组纹理坐标的小数部分
    o.color = frac(v.texcoord1);
    if(any(saturate(v.texcoord1) - v.texcoord1))
    {
    o.color.b = .5;
    }
    o.color.a = 1;
    //可视化顶点颜色
    o.color = v.color;

    return o;
    }

    fixed4 frag(v2f i) : SV_Target
    {
    return i.color;
    }

    ENDCG
    }

2.1.4.UnityShader的整洁

  • 尽可能使用小精度数值，float 32位，half 16位，fixed 11位
  • 慎用分支和循环语句
  • 避免不必要的计算
  • 不可以除以0

2.2.Unity中的基础光照

2.2.1.光源

  • 辐射度：用于量化光，计算方式是通过计算在垂直于光源方向l的单位面积上单位时间内穿过的能力
  • 对于和物体表面不垂直的光线，可以使用l和表面法线n进行计算，也就是lncosθ

2.2.2.吸收和散射

  • 散射：改变光线方向，有折射或者投射（这是内部），还有反射（这是外部）
  • 吸收：改变光线密度和颜色，不改变方向
  • 两种反射
    高光反射：表示物体表面如何反射光线
    漫反射：用于表示多少光线会被折射、吸收和散射

2.2.3.着色

  • 着色：根据材质属性（漫反射属性等）、光源信息（光源方向、辐射度等），使用一个等式去计算沿着摸个观察方向的出射度的过程。而这个等式就称为光照模型
  • 着色频率：
    逐顶点着色（高洛德着色）：在顶点着色器中，在每个顶点上计算光照，然后通过线性插值进行图元内部渲染，可能导致顶点处颜色是最高的
    逐像素着色（Phong着色）：在片元着色器中，以每个像素为基础，得到它的法线，然后进行光照模型计算

2.2.4.标准光照模型

  • 标准光照模型将进入到摄像机内的光分为4部分：
  1. 自发光：直接从物体进入到摄像机的光线
  2. 高光反射：用于计算那些沿着完全镜面反射方向被反射的光线，这可以让物体显得有光泽
  3. 漫反射：被物体表面随机散射到各个方向的辐射度
  4. 环境光：所有其他光线

2.3.基础纹理

  • 纹理实现，就是给每一个顶点分配一个uv坐标，让这个顶点知道自己所要采样的纹理。
  • UV 坐标只是映射纹理的二维坐标，用于从纹理中采样颜色。如果没有纹理，UV 坐标本身不会影响颜色。

2.3.1.单张纹理

  • 纹理Shader贴图
  • 纹理属性：
  1. 纹理类型（Texture Type）
  2. Wrap Mode：当纹理坐标超过[0, 1]范围，应该如何表示。常见有Wrap、Mirror、Repeat
  3. Filter Mode：决定了纹理被拉伸时，采用哪种滤波。引入了mipmap技术（空间换时间），提前用滤波技术处理来得到很多更小的图像，形成图像金字塔
  4. 纹理的最大尺寸和纹理模式：如果超过了Max Size，那么就会把该纹理缩放为该最大尺寸；纹理模式精度越高，占用内存越大，非必要时，一般都是压缩

2.3.2.凹凸映射

  • 纹理可以用于实现凹凸映射，原理是使用一张纹理改变物体表面法线
  1. 高度纹理：使用一张高度图来存储强度值，用于表示模型表面局部的海拔高度，越深就向内凹陷，越浅就向外凸出
  2. 法线纹理：法线纹理存储的是表面法线方向，分量范围在[-1, 1]，所以需要映射到像素分量，也就是[0, 1]。
    pixel = normal/2 + 0.5， normal = 2 pixel - 1
    有两种存储空间，一个是模型空间，还有一个是切线空间。切线空间以法线为z轴，顶点切线为x轴，而法线和切线的叉积得到副切线，也就是y轴。
    两者分别优缺点：
    a. 模型空间：直观，实现简单，计算更少；缝合处和纹理边界可见的突变和缝隙较少
    b. 切线空间：自由度高，用到其他模型也可以得到合理结果；可以进行UV动画；可以重用法线纹理；可以压缩，因为可以只存储xy值，z用叉积得出
  • 实现时根据光照计算所使用的空间有两种情况：
  1. 在切线空间进行光照计算
    需要将光照方向、视角方向变换到切线空间里面，然后进行计算。
    效率上，切线空间计算更优，因为在顶点着色器里面就直接将光照方向和视角方向进行变换
  2. 在世界空间进行光照计算
    需要将采样得到的法线变换到世界空间，然后进行计算
    通用性上，世界空间计算更优，因为一些其他的例如CubeMap进行环境映射时，需要法线方向变换到世界空间，这样就可以复用

2.3.3.渐变纹理（Ramp Texture）

  • 用一张纹理图存储渐变信息，从而控制漫反射光照的结果

2.3.4.遮罩纹理

  • 通过得到遮罩纹理的纹素值，然后使用其中某个或者几个通道的值，与某种表面属性进行相乘。如果纹素为0，那么该属性就不会影响到我们的模型表面。从而达到更加精确（像素级别）控制模型表面的各种性质

2.4.透明效果

2.4.1.基础实现

  • 透明通道：开启透明混合后，在颜色值和深度值之外，还有一个透明度，1为完全不透明，0为完全透明
  • 在正常情况下，开启深度写入和深度缓存，可达到遮挡效果，
  • 透明效果实现的两种方法：
  1. 透明度测试：如果透明效果大于阈值，就直接当作非透明物体处理；如果小于阈值，就当作透明物体直接完全透明。这种方法就是1和0的区别
  2. 透明度混合：需要关闭深度写入，会使用当前片元的透明度作为混合因子，与已经存储在颜色缓存中的颜色混合，得到新的颜色。

2.4.3.ShadeLab的混合命令

  • 混合不仅仅可以用于透明度混合
  • 混合实现原理：片元着色器产生一个颜色的时候，可以选择与颜色缓存中的颜色进行混合。
    因此混合就和两个操作数相关：源颜色（source color，片元着色器产生的颜色）和目标颜色（destination color，颜色缓冲中读取的颜色值）。混合后的输出颜色，也就是O ,他会重新写入颜色缓冲中。包含的是RGBA
    输出混合公式：Orgb = SrcFactor * Srgb + DstFactor * Drgb；Oa = SrcFactorA * Sa + DstFactor * Da
  • Blend的开启，如果是Blend Off以外的命令，都是默认打开了Blend On
  • 有一些常见混合操作，如图

2.4.4.双面渲染的透明效果

  • 双面渲染：可以看到物体内部结构
  • 通过Cull操作，Cull意味着剔除效果，默认剔除了背面（背向相机面），可以是Cull Back/Front/Off
  • 透明度测试的双面渲染：直接在Tags后面添加 Cull Off
  • 透明度混合的双面渲染：在两个Pass里面分别进行渲染剔除，按着一定顺序，在第一个Pass里面先渲染背面，也就是剔除前面，第二个里面渲染前面，也就是剔除背面

3.中级篇

3.1.更复杂的光照

3.1.1.渲染路径

  • 决定光照如何应用到Unity Shader中。也就是说，给每个Pass响应的渲染路径，Unity才知道要使用的光源和处理后的光照信息是哪些。
  • 三种渲染路径：
  1. 前向渲染路径（Forward Rendering Path）
  2. 延迟渲染路径（Deffered Rendering Path）（已替换为新的延迟渲染路径）
  3. 顶点照明渲染路径（Vertex Lit Rendering Path）（已弃用）

  • 如果整个项目只使用一个渲染路径，那么Edit->Project Settings->Player->Other Settings->Rendering Path，在里面选择项目所需要的渲染路径；如果不同相机想要用不同的渲染路径，那么在相机里面可以选择Rendering Path

  • 前向渲染路径：渲染该对象的渲染图元，然后进行两个缓冲区的计算，深度缓存判断一个片元是否可见，可见的话就将颜色缓冲的颜色值更新。如果场景里面有N个物体，每个物体都受到M个光照，那么就需要N * M 个Pass的计算
    一个Pass不仅可以用来计算逐像素光照，还可以计算逐顶点等其他光照。
    前向渲染路径有三种方式（选择取决于光的类型和渲染模式）：

  1. 逐顶点处理：最多四个光源
  2. 逐像素处理：一定数目的光源，最亮的平行光、Important光源
  3. 球谐函数处理（SH）：剩下的光源，渲染模式为Not Important的光源
    前向渲染的两种Pass
  4. BasePass（只进行一次计算）：
  5. AdditionalPass：
    前向渲染的问题：当场景包含大量光源时，会进行大量计算，多光源影响的区域就会相互重叠，导致需要对每一个物体进行多个Pass计算。
  • 顶点照明渲染路径：仅仅使用逐顶点的方法来计算光照，而不能使用逐像素的方法，硬件要求最低，运算性能最高
  • 延迟渲染路径：延迟渲染会使用额外的缓冲区，也就是G-Buffer（Geometry）,存储了我们关心的表面信息，如表面的法线、位置、用于光照计算的材质属性
    延迟渲染路径的两个Pass：(与光源数量无关，只与屏幕空间大小有关)
  1. 深度缓冲技术，将相关信息写入G-Buffer中
  2. 利用G缓冲区中的各个片元信息，进行真正的光照计算
    延迟渲染的缺点：
  3. 不支持真正抗锯齿
  4. 不能处理半透明物体
  5. 对显卡有一定要求

3.1.2.Unity的光源类型

  • 不同光源类型
  1. 平行光：无确定位置、方向恒定单一、颜色单一、无光照衰减
  2. 点光源：有位置属性、方向属性（目标点位置减去点光源位置）、颜色单一、强度可调、有光照衰减（可以由函数定义）
  3. 聚光灯：有位置属性、方向属性（同点光源）、颜色单一、强度可调、有光照衰减（函数定义较点光源更为复杂，加一次是否在椎体内的判断）

一些CG内置函数

  1. saturate(x)
    x不论是float，float2，float3，都会被截取在[0, 1]范围内
  2. reflect(i, n)
    计算反射方向，其中i为入射方向，n为法线方向
  3. o.uv = TRANSFORM_TEX(v.texcoord, Texture);
    可以从局部坐标转换为空间坐标，并且将Texture贴到模型上
  4. tex2D(Texture, i.uv);
    根据UV坐标的映射，获取每个点的纹素
  5. clip(color.a - _Cutoff);
    用于深度测试，等同于if (color.a - _Cutoff < 0) discard;
