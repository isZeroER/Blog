1.基础篇

1.1.渲染流水线

  1. 应用阶段（CPU执行）
  • 准备场景数据：摄像机位置、视锥体、场景中模型、使用的光源；
    粗粒度剔除：把不可见物剔除；
    设置渲染状态：模型所要使用到的纹理、材质、shader
  • 应用阶段将渲染图元传递给几何阶段
  1. 几何阶段（GPU执行）
  • 对渲染图元进行处理，输出屏幕空间的二维顶点坐标、每个顶点对应的深度值、着色等相关信息，传递给光栅化阶段
  1. 光栅化阶段（GPU执行）
  • 根据几何阶段传递的数据进行屏幕像素绘制，渲染出最终图像

1.2.GPU与CPU通信

1.2.1.CPU（应用阶段）

  1. 把数据加载到显存中（HDD-RAM-VRAM）
  2. 设置渲染状态
  3. 调用DrawCall，也就是渲染命令（一般在几十到一百）

1.2.2. GPU流水线（几何阶段和光栅化阶段）

  • 几何阶段：
  1. 顶点着色器（Vetex Shader）：负责进行坐标变换和逐顶点光照
  2. 剪裁（Clipping）：将一些摄像机视野外的场景物体处理掉
  3. 屏幕映射（Screen Mapping）：将每个图元的xy坐标映射到屏幕坐标系，z坐标系不变。
  • 光栅化阶段：计算每个图元覆盖了哪些像素，以及为这些像素计算他们的颜色
  1. 三角形设置（Triangle Setup）：计算光栅化一个三角网格所需信息
  2. 三角形遍历（Triangle Traversal），又称扫描变换（Scan Conversion）：检查每个像素是否被一个三角网格覆盖，是则产生片元（fragment，包含很多状态信息的集合）。还会根据三角形设置输出的三个顶点信息对整个覆盖区域的像素进行插值
  3. 片元着色器（Fragment Shader）：输出一个或者多个颜色值，其中包含纹理采样等渲染技术
  4. 逐片元操作（Per-Fragment Shader）：通过多个测试来决定每个片元的可见性，并且把这个片元的颜色值与已经存储在颜色缓冲区中的颜色进行合并
    a. 模板测试：
    通过模板比较函数，判断模板缓冲区中该片元的模板值与参考值的大小关系来决定是否舍弃
    b. 深度测试：
    通过深度比较函数，判断该片元的深度值与已经存在于深度缓冲区的深度值的大小关系来决定是否舍弃
    c.合并：
    通过a、b测试后，看是否开启混合（Blender）模式，以判断是否混合还是直接覆盖。
  • 一些问题：
  1. 按正常的渲染管线，那么深度测试在管线的末端，如此前期可能有一些片元经过了一些列操作之后，在深度测试没有通过，那么浪费了很多性能
  2. 有一种Early-Z的技术，将深度测试提前到片元着色器之前可以带来很多性能提升，但是这样可能会与片元着色器的一些操作冲突。

1.2.3.CPU与GPU工作

  • CPU将Draw Call以及改变渲染状态命令发送到命令缓冲区，GPU从中读取命令。
  • 实际上GPU的执行速度很快，但是CPU的速度会比较慢，因为多Draw Call涉及到准备数据、状态和命令，还要检查当前渲染状态
  • 减少Draw Call的办法之一：批处理（Batching），可以对静态物体进行一次合并，避免用大量很小的网格，也避免使用过多材质

1.3.Unity Shader

1.3.1.Unity中的Shader

  1. Standard Surface Shader：包含了一个标准光照模型的表面着色器模板
  2. Unlit Shader：不包含光照（但包含雾效）的基本顶点/片元着色器
  3. Image Effect Shader：提供实现各种屏幕后处理效果的基本模板
  4. Compute Shader：利用GPU并行性来进行一些与常规渲染流水线无关的计算

1.3.2.Unity Shader的ShaderLab

  • 基础结构：
  • 表面着色器：与光源有很多交道
  • 顶点/片元着色器：更加灵活，且适合光照比较少的情况

1.4.MVP变换

  • 顶点着色器：负责进行坐标变换，通过矩阵的方式进行变换
  • 变换过程：
  1. 模型空间：也就是局部空间/对象空间
  2. 世界空间：通过模型变换，从模型空间到局部空间
  3. 观察空间：通过观察变换，从世界空间投射到摄像机的观察空间
  4. 裁剪空间：通过投影变换，将观察空间的顶点剪裁到视锥体当中
  5. 屏幕空间：通过屏幕映射将视锥体中点投射到屏幕空间当中
  • 其中只有观察空间，也就是相机空间是右手系，其他都是左手系

2.初级篇

2.1.简单Shader学习

2.1.1.Shader编写

  • 简单实现
  • 定义结构体
    Pass
    {
    CGPROGRAM
    #pragma vertex vert
    #pragma fragment frag

    #include “UnityCG.cginc”

    struct v2f
    {
    float4 pos:POSITION;
    fixed4 color:COLOR0;
    };

    v2f vert(appdata_full v)
    {
    v2f o;
    o.pos = UnityObjectToClipPos(v.vertex);

    //可视化法线
    o.color = fixed4(v.normal*.5 + fixed3(.5, .5, .5) , 1);
    //可视化切线
    o.color = fixed4(v.tangent.xyz * .5 + fixed3(.5, .5, .5), 1);
    //可视化副切线
    fixed3 binormal = cross(v.normal, v.tangent.xyz) * v.tangent.w;
    o.color = fixed4(binormal * .5 + fixed3(.5, .5, .5), 1);
    //可视化第一组纹理坐标
    o.color = fixed4(v.texcoord.xy, 0, 1);
    //可视化第二组纹理坐标
    o.color = fixed4(v.texcoord1.xy, 0, 1);
    //可视化第一组纹理坐标的小数部分
    o.color = frac(v.texcoord);
    if(any(saturate(v.texcoord) - v.texcoord))
    {
    o.color.b = .5;
    }
    o.color.a = 1;
    //可视化第二组纹理坐标的小数部分
    o.color = frac(v.texcoord1);
    if(any(saturate(v.texcoord1) - v.texcoord1))
    {
    o.color.b = .5;
    }
    o.color.a = 1;
    //可视化顶点颜色
    o.color = v.color;

    return o;
    }

    fixed4 frag(v2f i) : SV_Target
    {
    return i.color;
    }

    ENDCG
    }

2.1.4.UnityShader的整洁

  • 尽可能使用小精度数值，float 32位，half 16位，fixed 11位
  • 慎用分支和循环语句
  • 避免不必要的计算
  • 不可以除以0
